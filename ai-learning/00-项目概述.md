## 项目运行流程简述
Query 优化→双路召回→Reranker 重排→动态时效性排序→专业回答生成
1. FastAPI 微服务接收用户口语化输入 → 
2. Query 口语转专业术语优化 → 
3. 调用 Qwen3-Embedding/bge 做向量化处理 → 
4. 双路召回 → 
    - 用向量结果去 Milvus 做向量相似性检索
    - 解析专业 Query 去 Neo4j 做知识图谱实体关系检索 
5. 召回结果结合 MongoDB 原始文档提取完整内容 → 
6. Qwen3-Reranker 重排得到精准上下文 → 
7. 交给 Qwen 大模型生成专业合规回答 → 
8. FastAPI 微服务返回最终结果

```bash
│   └── main.py         # 接口主文件
├── config/             # 配置文件模块
│   └── config.py       # 所有配置参数（数据库、模型、RAG）

├── model/              # 模型调用模块
│   ├── embedding.py    # Embedding模型调用
│   ├── llm.py          # Qwen大模型调用
│   └── reranker.py     # Qwen3-Reranker调用
├── rag/                # RAG核心逻辑模块
│   ├── basic_rag.py    # 基础RAG流程
│   └── enhanced_rag.py # 增强版RAG（双路召回+重排+动态排序）
├── data/               # 数据目录（保险文档、日志文件）
│   ├── docs/           # 原始保险文档

│   └── docker-compose.yml # 三大数据库一键部署配置
└── requirements.txt    # 项目所有依赖库
```
