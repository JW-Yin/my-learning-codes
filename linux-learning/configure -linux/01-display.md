# 安装缩放优化工具
sudo apt install gnome-tweaks

# 设置方法：
# 1. 打开"设置"->"显示器"
# 2. 将缩放比例设为125%或150%
# 3. 在GNOME Tweaks->"字体"中，将缩放因子设为1.25

你这个思路是对的！但“系统缩放100% + 字体1.25倍缩放”只能解决**字体大小**，没法让应用的按钮、菜单等界面元素同步缩放，所以还是会出现“显示不全”。结合你已装 `gnome-tweaks` 的情况，我给你**优化后的精准步骤**，让系统+应用全量适配1.25倍缩放，彻底解决显示问题：

### 核心逻辑：先开系统分数缩放（让界面元素缩放），再用Tweaks微调字体（让文字清晰）
#### 步骤1：先启用系统125%分数缩放（关键！之前没开的话必须做）
如果现在系统缩放还是100%，先执行命令开启分数缩放，让系统支持125%选项：
```bash
# 开启分数缩放支持（无论Wayland/Xorg都要执行）
gsettings set org.gnome.mutter experimental-features "['scale-monitor-framebuffer']"
```
重启电脑后，打开「设置」→「显示」：
- 此时应该出现「125%」选项（之前没有的话现在会有），**选择125%**（和你Tweaks的字体缩放一致，避免双重缩放模糊）；
- 确认分辨率是原生最高（如2880×1800）。

#### 步骤2：用GNOME Tweaks微调字体（让文字更清晰，避免模糊）
打开「优化」（就是你装的 gnome-tweaks）：
1. 左侧选择「字体」；
2. 把「缩放因子」设为 **1.0**（重点！因为系统已经开了125%缩放，这里再设1.25会双重缩放，导致文字模糊）；
3. 按需调整字体大小（比如“窗口标题”设为11，“界面字体”设为10），预览清晰即可。

#### 步骤3：修复仍显示不全的应用（针对性适配）
如果还有个别应用（比如Qt、Electron类）按钮截断，按类型单独设置：
##### ① Qt应用（如Wiliwili、部分开发工具）
终端启动时加缩放参数，比如启动Wiliwili：
```bash
QT_SCALE_FACTOR=1.25 flatpak run cn.xfangfang.wiliwili
```
（想永久生效的话，把命令添加到启动器快捷方式里，后面可以教你）

##### ② Electron应用（除了VS Code，其他如部分客户端）
```bash
ELECTRON_FORCE_DEVICE_SCALE_FACTOR=1.25 应用名
```

##### ③ GTK老应用（如部分系统工具）
打开「优化」→「字体」→ 把「Hinting」设为「中等」，「抗锯齿」设为「RGBA」，文字会更清晰，界面也会适配更好。

### 验证效果
调整后打开之前显示不全的软件（比如B站客户端、Flameshot），应该能看到：
- 按钮、菜单完整显示，不会被截断；
- 文字清晰不模糊，大小适中；
- 整个UI布局协调，不再有“半瓣显示”的情况。

### 常见问题排查
1. 重启后系统缩放又变回100%？  
   执行 `gsettings set org.gnome.desktop.interface scaling-factor 0`，再重启即可锁定分数缩放。
2. 部分应用模糊？  
   把系统缩放改成150%，Tweaks字体缩放设为1.0，适配性会更好（高分辨率屏幕150%更通用）。

要不要我帮你设置「永久生效的应用缩放参数」？比如把Wiliwili、Flameshot的启动命令改成带缩放参数的，不用每次终端输入，直接点图标就能正常显示~
