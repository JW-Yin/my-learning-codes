SSH 全称是 **Secure Shell（安全外壳协议）**，你可以把它理解成一条「加密的专属管道」，专门用来在互联网这种不安全的环境里，安全地连接和管理你的 Linux 服务器。

---

### 为什么我们必须用 SSH 连接 Linux 服务器？
在没有 SSH 之前，人们用 `Telnet` 等工具远程连接服务器，但这些工具是**明文传输数据**的——你的密码、执行的命令，甚至传输的文件，都像写在明纸上一样，很容易被黑客截获。

而 SSH 解决了这个核心问题：
1.  **全程加密传输**
    所有在你本地电脑和服务器之间传输的数据（包括登录密码、命令、文件），都会被高强度加密，黑客即使截获了数据，也无法破解成明文。
2.  **身份验证更安全**
    支持「密码登录」和更安全的「密钥对登录」，后者几乎不可能被暴力破解。
3.  **功能全面**
    不只是登录，还可以用它加密传输文件（`scp`/`sftp` 命令）、建立加密隧道，甚至管理服务器上的后台服务。

---

### 用一个简单的类比理解 SSH
你可以把 SSH 想象成：
- 你要给远方的服务器「发消息」（执行命令、登录），直接发会被路人（黑客）偷看。
- SSH 就像给你和服务器之间建了一条「加密管道」，所有消息都在管道里传输，路人看不到管道里的内容，也无法篡改消息。

---

### 用 CRT 连接服务器的本质
你用 SecureCRT（CRT）连接阿里云服务器时，CRT 就是一个 **SSH 客户端工具**，它帮你和服务器建立加密连接：
1.  你输入服务器公网 IP、账号密码（或密钥）。
2.  CRT 通过 SSH 协议和服务器的 `sshd` 服务（SSH 服务端）建立加密通道。
3.  连接成功后，你在 CRT 里敲的命令、服务器返回的结果，都会通过这个加密通道传输。

---

### 为什么它是云服务器的标配？
阿里云等云厂商的 Linux 服务器默认只开启 SSH 服务（端口 22），禁止了不安全的 `Telnet`，就是因为：
- 云服务器直接暴露在公网，必须用加密协议保护登录和管理操作。
- SSH 是行业公认的安全标准，几乎所有 Linux 系统都原生支持。
