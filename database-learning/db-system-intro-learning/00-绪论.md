# 《数据库系统概论（第5版）》第一章 绪论大纲

## 第1章 绪论

### 1.1 数据库系统概述

#### 1.1.1 数据库的4个基本概念
- **数据（data）**
  - 数据库存储的基本对象
  - 广义理解：数字、文本、图形、图像、音频、视频、记录等
  - 描述事物的符号记录
  - 语义与数据不可分
- **数据库（DataBase，DB）**
  - 长期存储在计算机内的有组织、可共享的大量数据集合
  - 特点：永久存储、有组织、可共享
- **数据库管理系统（DBMS）**
  - 位于用户与操作系统之间的数据管理软件
  - 功能：数据定义、组织存储管理、数据操纵、事务管理、数据库建立维护等
- **数据库系统（DBS）**
  - 由数据库、DBMS、应用程序和数据库管理员组成的系统
  - DBA职责：全面管理数据库系统

#### 1.1.2 数据管理技术的产生和发展
- **人工管理阶段（20世纪50年代中期前）**
  - 特点：数据不保存、应用程序管理数据、数据不共享、数据不具有独立性
- **文件系统阶段（20世纪50年代后期到60年代中期）**
  - 特点：数据长期保存、文件系统管理数据、数据共享性差冗余度大、数据独立性差
- **数据库系统阶段（20世纪60年代后期以来）**
  - 特点：数据结构化、共享性高冗余度低、数据独立性高、数据由DBMS统一管理控制

#### 1.1.3 数据库系统的特点
- **数据结构化**
  - 整体数据结构化，数据之间具有联系
- **数据的共享性高、冗余度低且易扩充**
  - 数据面向整个系统，可被多用户多应用共享
- **数据独立性高**
  - 物理独立性、逻辑独立性
- **数据由DBMS统一管理和控制**
  - 数据安全性保护、完整性检查、并发控制、数据库恢复

### 1.2 数据模型

#### 1.2.1 两类数据模型
- **概念模型（信息模型）**
  - 按用户观点对数据和信息建模，用于数据库设计
- **逻辑模型**
  - 层次模型、网状模型、关系模型、面向对象数据模型等
  - 按计算机系统观点对数据建模，用于DBMS实现
- **物理模型**
  - 描述数据在系统内部的表示方式和存取方法

#### 1.2.2 概念模型
- **信息世界基本概念**
  - 实体（entity）、属性（attribute）、码（key）
  - 实体型（entity type）、实体集（entity set）、联系（relationship）
- **实体-联系方法**
  - E-R图描述概念模型
  - 实体间联系类型：一对一、一对多、多对多

#### 1.2.3 数据模型的组成要素
- **数据结构**
  - 描述数据库组成对象及对象间联系
- **数据操作**
  - 查询、更新（插入、删除、修改）操作
- **数据的完整性约束条件**
  - 实体完整性、参照完整性、用户定义完整性

#### 1.2.4 常用的数据模型
- **层次模型（hierarchical model）**
- **网状模型（network model）**
- **关系模型（relational model）**
- **面向对象数据模型**
- **对象关系数据模型**
- **半结构化数据模型**

#### 1.2.5 层次模型
- **数据结构**
  - 树形结构表示实体及联系
  - 条件：有且仅有一个根结点、其他结点有且仅有一个双亲结点
- **数据操纵与完整性约束**
  - 查询、插入、删除、更新操作
  - 插入删除操作受完整性约束
- **优缺点**
  - 优点：结构简单清晰、查询效率高、完整性支持好
  - 缺点：表示非层次联系困难、插入删除限制多、查询需通过双亲结点

#### 1.2.6 网状模型
- **数据结构**
  - 允许结点无双亲或有多于一个双亲
  - 更直接描述现实世界
- **数据操纵与完整性约束**
  - 支持记录码概念、保证一对多联系、支持约束条件
- **优缺点**
  - 优点：直接描述现实世界、存取效率高
  - 缺点：结构复杂、DDL和DML复杂、用户需了解系统细节

#### 1.2.7 关系模型
- **数据结构**
  - 建立在严格数学概念基础上
  - 二维表表示关系
  - 关系（表）、元组（行）、属性（列）、码、域、关系模式
- **数据操纵与完整性约束**
  - 查询、插入、删除、更新操作
  - 实体完整性、参照完整性、用户定义完整性
- **优缺点**
  - 优点：概念单一、结构简单清晰、数据独立性强
  - 缺点：查询效率可能不如格式化模型

### 1.3 数据库系统的结构

#### 1.3.1 数据库系统模式的概念
- **型（type）与值（value）**
  - 型：对数据的结构和属性的说明
  - 值：型的具体赋值
- **模式（schema）与实例（instance）**
  - 模式：数据库中全体数据的逻辑结构和特征描述
  - 实例：模式的一个具体值
  - 模式相对稳定，实例相对变动

#### 1.3.2 数据库系统的三级模式结构
- **模式（逻辑模式）**
  - 数据库中全体数据的逻辑结构和特征描述
  - 所有用户的公共数据视图
- **外模式（用户模式）**
  - 数据库用户能看见和使用的局部数据的逻辑结构和特征描述
  - 模式的子集，保证数据库安全性
- **内模式（存储模式）**
  - 数据物理结构和存储方式的描述
  - 数据在数据库内部的组织方式

#### 1.3.3 数据库的二级映像功能与数据独立性
- **外模式/模式映像**
  - 定义外模式与模式之间的对应关系
  - 保证数据的逻辑独立性
- **模式/内模式映像**
  - 定义数据全局逻辑结构与存储结构之间的对应关系
  - 保证数据的物理独立性

### 1.4 数据库系统的组成

#### 1.4.1 硬件平台及数据库
- 足够大的内存存放系统核心模块
- 足够大的磁盘存放数据库
- 较高的通道能力提高数据传送率

#### 1.4.2 软件
- 数据库管理系统（DBMS）
- 支持DBMS运行的操作系统
- 高级语言及其编译系统
- 应用开发工具
- 数据库应用系统

#### 1.4.3 人员
- **数据库管理员（DBA）**
  - 决定数据库信息内容和结构
  - 决定存储结构和存取策略
  - 定义安全性和完整性约束
  - 监控数据库使用和运行
  - 改进和重组重构数据库
- **系统分析员和数据库设计人员**
  - 系统分析员：需求分析和规范说明
  - 数据库设计人员：数据确定和模式设计
- **应用程序员**
  - 设计和编写应用系统程序模块
- **用户（最终用户）**
  - 偶然用户、简单用户、复杂用户

### 1.5 小结
- 数据库基本概念
- 数据管理技术发展：人工管理、文件系统、数据库系统
- 数据模型：概念模型、逻辑模型、物理模型
- 关系模型：最重要数据模型，本书重点
- 数据库三级模式和两级映像
- 数据库系统组成：硬件、软件、人员
